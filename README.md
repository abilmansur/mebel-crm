# Мебель CRM — MVP

CRM для мебельных цехов: единый инбокс заявок (WhatsApp/Telegram/звонки/сайт) + канбан-доска заказов
+ калькулятор сметы по материалам (ЛДСП/МДФ), фурнитуре и комментарию к заказу.
Каждый цех — отдельный аккаунт с изолированными данными через Supabase Auth + Row Level Security.

## Локальный запуск (без Supabase, на демо-данных)

```bash
npm install
npm run dev
```

Открой http://localhost:3000 — приложение работает на встроенных демо-данных
из `lib/demoData.ts`, без входа и без настройки базы. Удобно, чтобы сразу показать прототип клиенту.

## Подключение реальной базы и авторизации (Supabase)

1. Создай проект на [supabase.com](https://supabase.com) (бесплатный тариф достаточен для старта)
2. В SQL Editor выполни содержимое `supabase/schema.sql` — создаст таблицы и политики RLS
3. В Authentication → Providers убедись, что включён Email-провайдер
4. Для быстрого тестирования (без проверки почты): Authentication → Settings → выключи
   "Confirm email" — тогда после регистрации сразу будет сессия. Для продакшена лучше оставить включённым
5. Скопируй `.env.local.example` в `.env.local` и вставь `Project URL` и `anon public key` из
   Settings → API в Supabase
6. Перезапусти `npm run dev`

## Как это работает

- При первом заходе без сессии пользователя перекидывает на `/login`
- На `/login` можно зарегистрировать новый цех (email + пароль + название цеха) или войти в существующий
- При регистрации автоматически создаётся `workspace` в базе и засеваются стартовые материалы
  (ЛДСП 16мм, ЛДСП 25мм, МДФ) — см. `lib/workspace.ts`
- RLS в `supabase/schema.sql` гарантирует, что цех A никогда не увидит заказы цеха Б,
  даже если оба используют одно и то же Supabase-приложение

## Деплой на Vercel

```bash
npm install -g vercel
vercel
```

Добавь переменные окружения (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`,
`NEXT_PUBLIC_CURRENCY_SYMBOL`) в Settings → Environment Variables на vercel.com,
либо через `vercel env add <имя>`.

## Что уже готово

- Регистрация/вход по email через Supabase Auth, свой workspace на каждый цех (`app/login/page.tsx`, `lib/workspace.ts`)
- Страница профиля: имя, телефон, смена пароля, выбор языка (Рус/Қаз/Eng) и темы (Дневная/Ночная) — язык и тема живут только здесь, не в шапке (`app/profile/page.tsx`, `lib/ThemeContext.tsx`)
- Тёмная тема на CSS-переменных — переключается глобально без доработки каждого компонента (`app/globals.css`, `tailwind.config.ts`)
- Аналитика: всего сделок, выручка по успешным заказам, конверсия, средний чек, воронка по этапам (`components/Analytics.tsx`, `lib/analytics.ts`)
- Мультиязычность интерфейса, выбор сохраняется в браузере (`lib/i18n.ts`, `lib/LanguageContext.tsx`)
- Row Level Security — изоляция данных между цехами на уровне базы (`supabase/schema.sql`)
- Единый инбокс заявок с AI-подсказкой ответа (`components/Inbox.tsx`)
- Канбан-доска: Заявка → Замеры → Согласовано → Производство → Доставка → Завершено, с автопометкой Успешно/Не успешно (`components/Board.tsx`)
- Калькулятор сметы: материал (площадь + кромка + наценка) + фурнитура с количеством + комментарий + телефон/адрес/даты замера и доставки (`lib/calculator.ts`, `components/OrderModal.tsx`)
- Настройки материалов: правка цены существующих + добавление новых материалов (`components/MaterialSettings.tsx`)
- Favicon с буквой «M» (`app/icon.svg`), широкий контейнер на больших экранах вместо зажатой колонки
- Адаптивная вёрстка под телефон, в т.ч. переработанный блок фурнитуры

## Что нужно доделать перед продажей нескольким цехам

1. **Реальный приём сообщений.** Таблица `inbox_messages` сейчас заполняется вручную —
   нужны вебхуки от Telegram Bot API и WhatsApp Business API.
2. **AI-подсказки по-настоящему.** Поле `ai_suggestion` сейчас статичное — интеграция с
   Claude/OpenAI API добавляется отдельным API-роутом (`app/api/ai-suggest/route.ts`), не в этом MVP.
3. **Оплата подписки.** Stripe/локальный эквайринг — на первых 3-5 клиентах можно выставлять счёт вручную.
4. **Уведомления о просрочке.** Поле `overdue` сейчас ставится вручную — нужен cron
   (Supabase Edge Function по расписанию), сравнивающий `due_date` с текущей датой.
5. **Восстановление пароля / смена email** — стандартные Supabase Auth флоу, не добавлены в MVP-интерфейс,
   но доступны через `supabase.auth.resetPasswordForEmail()` при необходимости.
6. **Перевод покрывает основной интерфейс** — часть плейсхолдеров в полях ввода намеренно оставлена
   на русском как разумный дефолт для первых клиентов; при желании дополняется в `lib/i18n.ts`.
7. **Удаление материала** пока не реализовано — только добавление и редактирование цены.
